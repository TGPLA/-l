# 深度复盘存档

## 问题：EPUB 导入弹窗点击无反应（Chrome 浏览器）

**时间**：2026-03-20
**状态**：未解决

---

## 核心矛盾总结

### 问题描述
在 Chrome 浏览器中，点击 EPUB 导入弹窗的文件选择区域，无法触发文件选择器。Trae 内置浏览器正常，但 Chrome 无反应，控制台无报错。

### 为什么过去几轮尝试都失败了？

**尝试 1：`<div onClick={() => fileInputRef.current?.click()}>`**
- 失败原因：Chrome 安全策略阻止通过 JS 触发的文件选择器，必须由用户"真实交互"触发

**尝试 2：`<label>` 包裹 `<input type="file">`**
- 失败原因：内部 SVG 和文字元素可能阻止了点击事件冒泡

**尝试 3：`<label htmlFor="epub-file-input">` 标准 HTML 方式**
- 失败原因：未知，理论上应该可行，但 Chrome 仍然无反应

**尝试 4：`<label htmlFor>` + onClick 手动触发双重保障**
- 失败原因：未知，可能存在其他干扰因素

### 深度分析

1. **事件冒泡问题**：外层遮罩层有 `onClick={handleClose}`，虽然有 `e.stopPropagation()`，但可能仍有问题

2. **React 合成事件问题**：React 的合成事件系统可能与原生事件有冲突

3. **浏览器差异**：Trae 内置浏览器（可能是 Electron 或特殊环境）与标准 Chrome 行为不同

4. **可能的遗漏**：
   - 弹窗的 z-index 层级问题
   - 某些 CSS 属性（如 `pointer-events`）可能被意外设置
   - React 严格模式下的事件处理问题
