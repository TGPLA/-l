# 失败复盘存档报告

## 问题概述

**问题描述**：EPUB 导入时封面数据无法保存到数据库，报错 "Data too long for column 'cover_url'"

**触发时间**：2026-04-01

## 核心矛盾分析

### 为什么过去几轮尝试都失败了？

**根本原因**：后端服务连接的是**远程数据库**（通过 SSH 隧道 `127.0.0.1:3307`），但迁移脚本是在**服务器本地数据库**（Docker 容器 `1Panel-mysql-OHb5`）上执行的。

**关键发现**：
- 后端配置：`DB_HOST=127.0.0.1`, `DB_PORT=3307`（SSH 隧道连接）
- 迁移执行：在 Docker 容器 `1Panel-mysql-OHb5` 上执行
- **这两个可能不是同一个数据库实例！**

### 逻辑冲突

1. **SSH 隧道连接**：本地后端通过 `127.0.0.1:3307` 连接到远程数据库
2. **Docker 容器**：迁移脚本在 Docker 容器内的 MySQL 上执行
3. **可能的情况**：
   - SSH 隧道连接的数据库和 Docker 容器内的数据库是**不同的实例**
   - 需要确认 SSH 隧道连接的具体数据库实例
