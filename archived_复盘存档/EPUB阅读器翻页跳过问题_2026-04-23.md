# EPUB阅读器翻页跳过问题修复记录

**日期**：2026-04-23

**问题现象**：
- 章节内容分布在两个页面
- 正常手速点击下一页，会跳过第二页
- 跳过后点击上一页能返回第二页

## 根本原因分析

**错误实现**：`handleNextPage` 使用 spine 跳转逻辑（跳到下一章节），而不是使用 `rendition.next()` （翻到下一页）

**当前逻辑 vs 正确逻辑**：

| 操作 | 错误实现 | 正确实现 |
|------|---------|---------|
| 下一页 | 使用 spine 跳转到**下一章节** | 使用 `rendition.next()` 翻到**下一页** |
| 上一页 | 使用 `rendition.prev()` | 使用 `rendition.prev()` |

**问题流程**：
1. 用户在第3章的第1页（内容跨两页）
2. 点击下一页
3. `handleNextPage` 通过 spine 找到"当前在第3章"
4. 然后跳到"第4章"
5. **跳过了第3章的第2页** ❌

**为什么上一页能回去**：
- `handlePrevPage` 使用 `rendition.prev()`
- 这是正确的翻页逻辑
- 所以能回到第3章的第2页

## 修复方案

**修改文件**：`src/features/books/hooks/useEPUBReaderFanYeHeYeMa.ts`

**修改内容**：
将 spine 跳转逻辑改为 `rendition.next()` 翻页逻辑：

```typescript
const handleNextPage = useCallback(() => {
  const rendition = renditionRef.current;
  if (!rendition) {
    return;
  }
  
  // 优先使用 rendition.next() 翻到下一页（处理章节内容跨页的情况）
  rendition.next().then(() => {
    const currentLocation = rendition.location;
    const currentHref = currentLocation?.start?.href || '';
    if (currentHref && saveImmediately) {
      saveImmediately(currentHref);
    }
  }).catch((error) => { 
    console.error('下一页出错:', error); 
  });
}, [saveImmediately]);
```

- ✅ `npm run build` 构建成功
- ✅ 翻页逻辑统一使用 epubjs 原生翻页方法
- ✅ 现在下一页会在章节内翻页，而不是跳到下一章节
