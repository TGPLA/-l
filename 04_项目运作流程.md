# 项目运作流程

> 用大白话解释项目运作逻辑，帮助小白理解

---

## 一、应用启动流程

```
用户打开网页
    ↓
main.tsx（入口文件）加载
    ↓
AppProvider.tsx 初始化全局状态
    ↓
检查用户登录状态（Go后端JWT认证）
    ↓
├── 未登录 → 显示登录页面（AuthPage.tsx）
└── 已登录 → 显示主应用内容
```

**关键文件：**
- `src/core/main.tsx` - 应用入口
- `src/infrastructure/hooks/AppProvider.tsx` - 全局状态管理
- `src/features/user/components/AuthPage.tsx` - 登录页面

---

## 二、用户登录流程

```
用户输入账号密码
    ↓
点击登录按钮
    ↓
调用 Go 后端认证 API
    ↓
后端验证账号密码，生成 JWT Token
    ↓
├── 成功 → 保存 JWT Token 和用户信息，跳转书架页
└── 失败 → 显示错误提示
```

**关键文件：**
- `src/shared/services/auth.ts` - 认证服务
- `backend/controllers/auth_login.go` - 登录控制器
- `backend/middleware/auth_middleware.go` - JWT 认证中间件

## 三、核心业务流程

### 3.1 书架管理

```
用户进入书架页面
    ↓
从数据库加载用户的书籍列表
    ↓
展示书籍卡片
    ↓
用户可以：搜索书籍、添加书籍、删除书籍
```

**关键文件：**
- `src/features/books/components/BookShelf.tsx` - 书架主页面
- `src/shared/services/database.ts` - 数据库操作

### 3.2 章节管理

```
用户进入书籍详情页

点击「章节管理」Tab

    ↓
用户可以：

├── 编辑章节（修改标题和内容）
```

**关键文件：**
- `src/features/books/components/ChapterView.tsx` - 章节视图入口
- `src/features/books/components/ChapterManager.tsx` - 章节管理组件
- `src/features/books/components/ChapterDetail.tsx` - 章节详情组件

### 3.3 AI 出题流程（重构）

```
用户进入章节详情页
    ↓
点击「AI 生成题目」按钮
    ↓
弹出选择框：
├── 基于整章内容出题
└── 基于段落内容出题（如果有段落）
    ↓

    ↓

    ↓

    ↓

    ↓

    ↓

    ↓

```

- `src/shared/services/aiService.ts` - AI 服务

- `backend/controllers/ai_evaluate.go` - AI 评价控制器
- `backend/controllers/ai_paragraph.go` - AI 段落处理控制器
- `backend/services/zhipu_core.go` - 智谱 AI 核心服务
- `backend/services/zhipu_questions.go` - 智谱 AI 出题服务
- `backend/services/zhipu_evaluate.go` - 智谱 AI 评价服务
- `backend/services/zhipu_paragraph.go` - 智谱 AI 段落服务

## 六、关键概念解释

### 6.1 什么是 Go 后端？
本项目使用 Go 语言（Gin 框架）构建后端 API：
- **认证**：JWT Token 认证机制
- **数据库**：MySQL 存储用户数据、书籍、章节、题目等
- **AI 服务**：调用智谱 AI 进行出题和评价

### 6.2 什么是智谱AI？
智谱AI（GLM-4）是本项目使用的国产大语言模型，用于：
- 根据章节内容生成问题
- 评判用户答案
- 提供学习建议

### 6.3 什么是三层架构？
本项目代码按三层组织：
- **数据逻辑层**：hooks/、services/、api/ - 处理数据和业务逻辑
- **子组件层**：components/business/ - 可复用的UI组件
- **主视图层**：components/*.tsx - 页面主组件

### 6.4 前后端数据格式转换
后端使用 snake_case（如 `book_id`），前端使用 camelCase（如 `bookId`）。
前端服务层负责数据格式转换，确保类型安全。

## 八、常见问题

**Q: 登录失败怎么办？**
A: 检查后端服务是否正常运行，查看 JWT Token 是否有效

**Q: AI不回复怎么办？**
A: 检查智谱AI API Key 是否有效，查看网络请求是否成功

**Q: 数据不同步怎么办？**
A: 检查后端 API 是否正常，查看数据库连接状态

**Q: 章节管理功能无法使用？**
A: 确保后端 `/api/chapters` 接口正常工作，检查数据库 chapters 表是否存在
