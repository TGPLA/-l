# 数据库迁移指南 v6.0

## 问题描述

**错误信息**：
```
Error 1406 (22001): Data too long for column 'cover_url' at row 1
```

**根本原因**：
- `cover_url` 字段定义为 `VARCHAR(512)`，只能存储 512 个字符
- Base64 编码的封面图片通常需要几万到几十万字符
- 导致数据无法插入数据库

## 解决方案

将 `cover_url` 字段类型从 `VARCHAR(512)` 改为 `TEXT`（可存储 65535 字符）

## 迁移步骤

### 方法一：通过 MySQL 客户端执行（推荐）

#### 1. 确保 SSH 隧道已建立
```powershell
ssh -i "$env:USERPROFILE\.ssh\id_rsa" -f -N -L 3307:127.0.0.1:3306 root@linyubo.top
```

#### 2. 连接数据库
```powershell
mysql -h 127.0.0.1 -P 3307 -u root -p
# 输入密码：mysql_h8TCKC
```

#### 3. 执行迁移脚本
```sql
USE reading_reflection;
ALTER TABLE books MODIFY COLUMN cover_url TEXT COMMENT '封面图片 URL 或 base64 数据';
```

#### 4. 验证修改
```sql
DESCRIBE books;
```

应该看到 `cover_url` 字段类型为 `text`。

### 方法二：通过服务器直接执行

#### 1. SSH 登录服务器
```powershell
ssh root@linyubo.top
```

#### 2. 连接 MySQL
```bash
mysql -u root -p
# 输入密码
```

#### 3. 执行迁移脚本
```sql
USE reading_reflection;
ALTER TABLE books MODIFY COLUMN cover_url TEXT COMMENT '封面图片 URL 或 base64 数据';
```

## 迁移后验证

迁移完成后，重新导入 EPUB 文件，封面应该能正确保存和显示。

## 回滚方案（如需）

如果迁移后出现问题，可以回滚：
```sql
ALTER TABLE books MODIFY COLUMN cover_url VARCHAR(512) COMMENT '封面图片 URL';
```

**注意**：回滚后已存储的长封面数据会被截断。
