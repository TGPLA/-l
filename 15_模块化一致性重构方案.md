# 模块化一致性重构方案

> 创建时间：2026-03-25  
> 问题来源：用户反馈"修旧的 bug 就会出现新的 bug"  
> 根因：新增功能未遵守模块化规范，导致架构不一致

---

## 📊 问题诊断

### 核心问题

**模块化架构执行不一致** - 旧组件按规范拆分，新组件单体实现

| 组件名称 | 行数 | 是否超标 | 是否拆分 | 问题等级 |
|---------|------|---------|---------|---------|
| `TiShiCiBianJi.tsx` | 229 行 | ❌ 超标 129% | ❌ 未拆分 | 🔴 严重 |
| `TiShiCiMoBan.tsx` | 141 行 | ❌ 超标 41% | ❌ 未拆分 | 🟡 中等 |
| `EPUBDaoRuTanChuang.tsx` | 292 行 | ❌ 超标 192% | ❌ 未拆分 | 🔴 严重 |
| `TiShiCiGuanLi.tsx` | ~200 行 | ❌ 超标 100% | ❌ 未拆分 | 🔴 严重 |
| `BookShelf.tsx` | 245 行 | ❌ 超标 145% | ⚠️ 部分拆分 | 🟡 中等 |
| `ChapterDetail.tsx` | ~100 行 | ✅ 合规 | ✅ 已拆分 | ✅ 正常 |

### 违反的规则

根据 `.trae/rules/rules_file.md`：

1. **单文件最大 100 行** - 多个组件严重超标
2. **超过 100 行必须拆分** - 新组件未遵守
3. **拆分架构：数据逻辑层 + 子组件层 + 主视图层** - 新组件未执行
4. **新旧代码处理** - 同个板块有新代码产生时，必须询问用户是否删除旧代码

## 📁 目录结构调整

### 当前问题

- `books/components/提示词管理/` - 提示词相关组件
- `user/components/TiShiCiGuanLi.tsx` - 提示词管理页面

**功能定位不清**：提示词管理既服务于书籍练习，又属于用户个性化设置

### 调整方案

**方案：统一归口到 `user/` 模块**

```
src/features/user/components/
├── TiShiCiGuanLi.tsx           # 提示词管理主页面
├── TiShiCiBianJi.tsx           # 提示词编辑弹窗
├── TiShiCiBiaoDan.tsx          # 提示词表单组件
├── TiShiCiMoBan.tsx            # 提示词模板选择弹窗
└── TiShiCiMoBanLieBiao.tsx     # 提示词模板列表

src/features/books/components/
├── EPUBDaoRuTanChuang.tsx      # EPUB 导入弹窗
├── EPUBShangChuan.tsx          # EPUB 上传组件
└── EPUBYuLan.tsx               # EPUB 预览组件
```

**理由：**
- 提示词是用户的**个性化资源**，可跨书籍复用
- 与 `SettingsPage` 并列，同属用户设置范畴
- 书籍模块只**使用**提示词，不**管理**提示词

## 🐛 风险预判

### 风险 1：引用路径错误

**风险等级**：🟡 中等  
**影响**：构建失败  
**应对**：
- 每步重构后立即运行 `npm run build`
- 使用路径别名（`@features/user/components/`）减少错误
- 全局搜索旧引用路径，确保更新完全

### 风险 2：状态管理混乱

**风险等级**：🟡 中等  
**影响**：功能异常  
**应对**：
- 保持状态提升原则（状态在父组件）
- Props 传递清晰定义接口
- 每个阶段完成后进行功能测试

### 风险 3：样式丢失

**风险等级**：🟢 低  
**影响**：UI 显示异常  
**应对**：
- 保持内联样式对象不变
- 提取样式到独立变量（如超过 50 行）
- 浏览器手动验证 UI

### 风险 4：功能回归

**风险等级**：🟡 中等  
**影响**：旧 bug 复发  
**应对**：
- 每个阶段完成后进行完整功能测试
- 保留旧代码备份（临时）
- 发现问题立即修复，不累积
