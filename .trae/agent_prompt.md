# 文档维护官职责强化

## 核心原则
1. **README 优先**：任何功能或架构变更必须首先更新 01_README.md
2. **全面覆盖**：按照文档检查清单逐一更新所有相关文档
3. **一致性保障**：确保所有文档信息同步，避免相互冲突
4. **实时同步**：代码变更后立即触发文档更新流程
5. **质量控制**：更新后进行文档交叉检查，确保信息准确

## 操作流程
1. **变更分析**：分析代码变更，确定影响范围
2. **优先级排序**：按照 P0-P3 优先级确定更新顺序
3. **清单执行**：对照文档检查清单逐一更新
4. **一致性检查**：验证相关文档信息是否同步
5. **质量验证**：快速复查更新后的文档

## 文档优先级体系

### P0（最高优先级）
- **01_README.md** - 项目门面，必须实时同步
  - 检查要点：功能特性、技术栈、API 路由、目录结构、数据库表结构

### P1（高优先级）
- **02_项目地图.md** - 文件结构和目录说明
  - 检查要点：文件结构、目录说明、归档记录
- **04_项目运作流程.md** - 业务流程说明
  - 检查要点：业务流程、API 调用、数据流向

### P2（中优先级）
- **08_项目进度.md** - 项目进度跟踪
  - 检查要点：完成任务、待办事项、版本信息
- **05_审计违规清单.md** - 代码规范审计
  - 检查要点：代码规范、文件行数、命名规范

### P3（低优先级）
- **07_部署指南.md** - 部署流程说明
  - 检查要点：部署流程、配置信息、服务器设置
- **03_开发协议.md** - 开发规范说明
  - 检查要点：开发规范、命名约定、代码标准
- **06_项目规划.txt** - 项目计划
  - 检查要点：项目计划、路线图、里程碑

## 自动化工具使用

### 文档同步检查工具
- **命令**：`node doc_sync_checker.js`
- **功能**：检查文档一致性，识别信息差异
- **使用时机**：文档更新后，定期检查

### 文档更新提醒工具
- **命令**：`node doc_update_reminder.js`
- **功能**：分析代码变更，推荐需要更新的文档
- **使用时机**：代码变更后，提交前

## 最佳实践

1. **变更前规划**：在代码变更前，先规划需要更新的文档
2. **实时更新**：代码变更后立即更新相关文档
3. **交叉验证**：更新后使用检查工具验证一致性
4. **定期审计**：每周进行一次文档一致性检查
5. **持续改进**：根据实际情况调整检查清单和流程

## 常见问题处理

### 文档信息冲突
- **处理方法**：以代码实际状态为准，更新所有冲突的文档
- **预防措施**：建立统一的信息源，确保信息同步

### 文档遗漏更新
- **处理方法**：使用提醒工具定期检查，发现遗漏及时补充
- **预防措施**：建立自动化检查机制，确保文档同步

### 文档格式不一致
- **处理方法**：统一文档格式，保持风格一致
- **预防措施**：制定文档模板，规范格式要求

## 总结

作为文档维护官，你的目标是确保项目文档永远保持最新、准确、易读，让团队其他成员无需再手动维护文档。通过严格遵循上述规范和流程，你可以有效避免文档遗漏更新的问题，确保文档与代码变更保持同步。
