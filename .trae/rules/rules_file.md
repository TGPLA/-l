# 文件操作规范

## 修改文件前

1. 先查阅 `02_项目地图.md`，确认文件的功能定位
2. 检查文件是否已有 `// @审计已完成` 标记
3. 确认修改后文件行数不超过 100 行

## 删除文件前

1. 全局搜索确认无其他文件引用
2. 无引用的文件：物理删除
3. 同步更新 `02_项目地图.md` 的【归档区】

## 创建文件时

1. 必须包含中文头部注释
2. 必须添加 `// @审计已完成` 标记
3. 行数控制在 100 行以内
4. 遵循三层架构拆分原则
5. **无论来源**：从外部复制、参考他人代码、AI 生成等任何来源的文件，进入项目文件夹后必须遵守以上规则
6. **新旧代码处理**：同个板块有新代码产生时，必须询问用户是否删除旧代码，避免僵尸代码

## 命名规范

- 文件名：中文拼音（如 `ShuJiTiShiCiPeiZhi.ts`）
- 函数名：中文（如 `huoQuPeiZhi`）
- 变量名：中文（如 `shuMing`）
- 注释：中文

## 行数限制

- 单文件最大 100 行
- 超过 100 行必须拆分
- 拆分架构：数据逻辑层 + 子组件层 + 主视图层
