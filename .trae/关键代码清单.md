## 📋 使用说明

本文档记录项目中**绝对不能随意删除或修改**的关键代码段。

### 标记规范

在代码中使用以下标记：

```typescript
// @关键代码-不要随意删除 [简短功能描述]
// 原因：详细说明为什么这段代码不能删除
关键代码...
```

### 修改前检查

在删除或修改任何代码前，先检查：
1. 代码中是否有 `@关键代码` 标记
2. 本文档中是否有相关记录
3. 如果有，必须先了解原因，再考虑替代方案

## 📝 添加新关键代码流程

当你发现某段代码很关键时：

1. **在代码中添加标记**
   ```typescript
   // @关键代码-不要随意删除 [功能描述]
   // 原因：为什么不能删除
   ```

2. **在本文档中添加记录**
   - 文件位置
   - 功能描述
   - 不可删除原因
   - 标记位置（行号）

3. **提交变更**
   ```bash
   git add .
   git commit -m "docs: 添加关键代码标记和清单"
   ```

## ⚠️ 历史问题记录

### 2026-03-26 - 认证失败后页面不刷新

**问题描述：**
- 书架为空，后端链接不上
- 用户反馈"更新完好像后端就链接不上"

**根因分析：**
- `database.ts` 中 401 处理的 `window.location.reload()` 被误删
- 导致认证过期后只清除 token，不刷新页面
- AppProvider 仍认为用户已登录，API 请求持续失败

**修复方案：**
- 重新添加 `window.location.reload()`
- 添加 `@关键代码` 标记
- 创建本文档

**预防措施：**
- 建立关键代码标记机制
- 建立关键代码清单
- 建立修改前检查流程
