# 代码修改检查清单

> 文档创建时间：2026-03-26
> 最后更新：2026-03-26

---

## 📋 前言

本文档提供**代码修改前必须执行的检查清单**，防止类似"关键代码被误删"的问题再次发生。

**使用场景：**
- 删除代码前
- 修改核心逻辑前
- 重构代码前
- 任何可能影响现有功能的修改前

---

## ✅ 修改前检查清单

### 阶段 1：变更范围确认

- [ ] **我要修改哪些文件？**
  - 列出所有涉及的文件路径
  - 区分：新增文件 / 修改文件 / 删除文件

- [ ] **修改的目的是什么？**
  - 修复 Bug？
  - 新增功能？
  - 重构优化？
  - 清理代码？

- [ ] **影响范围有多大？**
  - 单一功能？
  - 多个模块？
  - 整个系统？

---

### 阶段 2：关键代码检查

- [ ] **目标文件中有关键代码标记吗？**
  - 搜索 `@关键代码` 关键字
  - 查看 [关键代码清单](./关键代码清单.md)
  - 如果有关键代码标记，必须先阅读原因

- [ ] **我要删除的代码有关键代码标记吗？**
  - 如果有，**停止！**
  - 先思考：为什么要删除？
  - 有没有替代方案？
  - 删除后会影响什么功能？

- [ ] **我要修改的代码是核心逻辑吗？**
  - 认证流程？
  - API 请求？
  - 状态管理？
  - 数据持久化？

---

### 阶段 3：删除代码确认

**如果要删除代码，必须回答以下问题：**

- [ ] **这段代码是做什么的？**
  - 用一句话描述功能
  - 如果说不清，**不要删除！**

- [ ] **为什么要删除？**
  - 代码过时了？
  - 有更好的实现？
  - 不再需要这个功能？
  - 还是只是"看起来没用"？

- [ ] **删除后会影响哪些功能？**
  - 列出所有可能受影响的功能
  - 检查调用链：谁调用了这段代码？
  - 检查数据流：数据从哪来？到哪去？

- [ ] **有测试验证删除是安全的吗？**
  - 单元测试？
  - 手动测试？
  - 如果没有测试，**不要删除！**

---

### 阶段 4：Git 操作检查

- [ ] **当前工作区是干净的吗？**
  - 运行 `git status`
  - 如果有未提交的修改，先提交或暂存

- [ ] **我是在正确的分支上吗？**
  - 建议在 feature 分支上修改
  - 不要直接在 main 分支上修改

- [ ] **我看过 Git diff 了吗？**
  - 运行 `git diff` 仔细检查每一行变更
  - 确认只有预期的变更
  - 没有意外的删除或修改

---

### 阶段 5：测试验证

- [ ] **构建成功吗？**
  - 运行 `npm run build`
  - 确保没有 TypeScript 错误
  - 确保没有 ESLint 错误

- [ ] **关键功能测试了吗？**
  - 登录/退出
  - 书架加载
  - 章节管理
  - 任何受影响的功能

- [ ] **浏览器控制台有错误吗？**
  - 打开开发者工具（F12）
  - 检查 Console 标签页
  - 检查 Network 标签页

---

## 🚨 危险信号

如果出现以下情况，**立即停止修改**：

### 1. 代码中有 `@关键代码` 标记
→ 不要删除或修改，先理解原因

### 2. 你说不清代码是做什么的
→ 不要删除，先花时间理解

### 3. 没有测试验证
→ 不要提交，先补充测试

### 4. Git diff 显示意外变更
→ 不要提交，先撤销意外变更

### 5. 你在"清理"代码
→ 谨慎！"清理"经常误删有用代码

---

## 📝 修改决策流程

```
开始修改
    ↓
我要删除代码吗？
    ├─ 是 → 有 @关键代码 标记吗？
    │         ├─ 是 → 停止！重新考虑
    │         └─ 否 → 完成检查清单 → Git diff → 测试 → 提交
    │
    └─ 否 → 有关键代码标记吗？
              ├─ 是 → 理解原因 → 小心修改 → 测试
              └─ 否 → 正常修改 → 测试 → 提交
```

---

## 🎯 快速检查（5 分钟）

如果时间紧迫，至少完成以下检查：

1. **搜索关键字**：在目标文件中搜索 `@关键代码`
2. **查看 Git diff**：运行 `git diff` 检查每一行
3. **构建测试**：运行 `npm run build`
4. **浏览器测试**：刷新页面，检查关键功能

## 💡 记住

**"删除比添加更危险！"**

- 添加代码：最坏情况是新功能有 Bug，可以回滚
- 删除代码：可能破坏现有功能，且很难发现是哪里删错了

**当你不确定时，就不要删！**
