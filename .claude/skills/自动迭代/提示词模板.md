# 自动迭代技能 - 提示词模板

> 使用这些提示词触发自动迭代技能的不同功能。

---

## 一、任务完成时触发迭代

### 场景1：功能开发完成
```
我完成了 [功能名称] 的开发，请运行自动迭代检查，
更新相关技能文档的模块导航表。
```

**示例：**
```
我完成了 EPUB阅读器夜间模式 的开发，请运行自动迭代检查，
更新前端开发技能的模块导航表。
```

### 场景2：代码重构完成
```
我重构了 [模块名称]，文件结构有变化，
请同步更新技能文档中的文件路径和模块分类。
```

**示例：**
```
我重构了 划线笔记模块，将 useHuaXianChuTi 拆分为多个Hook，
请同步更新前端开发技能中的 Hook 列表。
```

### 场景3：构建成功后自动触发
```
npm run build 成功了，请检查是否有需要同步到技能文档的变更。
```

## 二、Bug修复时触发迭代

### 场景1：修复已知Bug
```
我修复了 [Bug描述]，请：
1. 检查 Bug索引 中是否已有记录
2. 如果有，确认解决方案是否需要更新
3. 如果无匹配，创建新的复盘存档并更新索引
```

**示例：**
```
我修复了 划线在不同浏览器不同步 的问题，请：
1. 检查 Bug索引 中是否已有 B001 记录
2. 确认解决方案是否需要补充新方案
3. 更新测试调试技能的速查表
```

### 场景2：遇到重复Bug（智能复用）
```
我遇到了 [Bug描述]，请：
1. 先查 Bug索引 匹配历史方案
2. 如果匹配到，展示所有可用方案（包括废弃的作为参考）
3. 让我选择使用哪个方案
4. 如果方案有效，标记为"在用"；如果无效，创建新方案并存入
```

**示例：**
```
我遇到了 翻页时跳过页面 的问题，请：
1. 先查 Bug索引 匹配历史方案
2. 如果匹配到 B003，展示方案A（rendition.next）和方案B（混合方案）
3. 让我选择使用哪个方案
4. 如果方案A有效，确认状态为"在用"；如果需要调整，创建新方案C并存入
```

### 场景3：创建复盘存档后同步
```
我刚创建了复盘存档 [文件名]，请将其中的解决方案
提取并更新到 Bug索引 和对应技能的速查表中。
```

**示例：**
```
我刚创建了复盘存档 "EPUB导入封面解析失败_2026-04-29.md"，
请将其中的解决方案提取并更新到 Bug索引 和前端开发技能的速查表中。
```

## 五、Bug智能复用专用提示词

### 核心提示词（最常用）
```
我遇到了 [Bug描述]，请：
1. 先查 Bug索引 匹配历史方案
2. 展示所有匹配到的 Bug 及其解决方案（包括优先级和状态）
3. 如果匹配到多个，按优先级排序（primary > alternative > fallback）
4. 让我选择使用哪个方案，或提供新的解决方案
5. 修复后，更新 Bug索引（标记状态、添加新方案或创建新Bug条目）
```

### 快速匹配提示词
```
快速匹配Bug：[关键词1] [关键词2]
```

**示例：**
```

```

```
请展示 Bug [Bug ID] 的所有解决方案，包括：
- 每个方案的适用场景
- 优先级
- 当前状态
- 核心代码片段
```

**示例：**
```
请展示 Bug B001 的所有解决方案，包括：
- 每个方案的适用场景
- 优先级
- 当前状态

## 九、提示词使用技巧

### 1. 关键词提取
- 描述Bug时，尽量包含**技术关键词**（如：划线、同步、storage、rendition.next）
- 描述功能时，尽量包含**模块名称**（如：EPUB阅读器、书架、练习系统）

### 2. 上下文补充
- 如果知道相关文件路径，请一并提供
- 如果知道Bug ID，可以直接引用（如：B001、B003）

### 3. 明确需求
- 使用"请"开头，明确告诉AI要做什么
- 如果需要多步骤，使用数字列表（1. 2. 3.）

### 4. 反馈确认
- AI执行后，请确认结果是否符合预期
- 如果有误，及时纠正并更新技能文档

## 十、AI自动尝试失败处理提示词（用户极简版）

### 核心原则
**用户只需说3句话，其他全部由AI自动完成：**
1. "我遇到了..." → AI自动查索引、选方案、写代码
2. "不行" / "没解决" → AI自动记录失败、尝试下一个方案
3. "解决了" / "可以了" → AI自动记录成功、更新索引

### 场景1：AI第1次尝试失败（用户说"不行"）
```
用户：不行 / 没解决 / 还是不行

AI自动执行：
1. 记录AI第1次尝试失败到 Bug索引.AI尝试历史
2. 分析失败原因
3. 自动尝试备选方案（alternative）
4. 生成新代码给用户验证
```

**示例对话（简单否定）：**
```
用户：我遇到了划线在不同浏览器看不到的问题

AI：🔍 匹配到 Bug B001，自动尝试方案A（primary）...
    📝 已生成修复代码，请验证

用户：不行

AI：⚠️ 记录AI第1次尝试失败
    原因：方案A在当前场景无效
    🔄 自动尝试备选方案B（alternative）...
    📝 已生成新代码，请验证
```

**示例对话（带线索 - 条件线索）：**
```
用户：我遇到了划线在不同浏览器看不到的问题

AI：🔍 匹配到 Bug B001，自动尝试方案A（primary）...
    📝 已生成修复代码，请验证

用户：不行，刷新页面后划线就消失了

AI：🎯 检测到新线索！
    线索提取：
    - 触发条件：刷新页面
    - 可能原因：数据未持久化或渲染时机问题
    
    ⚠️ 记录AI第1次尝试失败（方案A不适用此场景）
    
    🔄 【基于线索调整】不直接尝试方案B，而是：
    1. 检查localStorage保存逻辑
    2. 添加页面加载时的划线重新应用
    3. 检查状态初始化顺序
    
    📝 已生成针对性修复代码，请验证
```

**示例对话（带线索 - 环境线索）：**
```
用户：我遇到了划线在不同浏览器看不到的问题

AI：🔍 匹配到 Bug B001，自动尝试方案A（primary）...

用户：不行，但是Safari可以，Chrome不行

AI：🎯 检测到新线索！

    - 环境差异：Safari ✅ vs Chrome ❌
    - 可能原因：浏览器API差异
    
    ⚠️ 记录AI第1次尝试失败（方案A在Chrome不适用）
    
    🔄 【基于线索调整】针对Chrome生成特定方案：
    1. 添加Chrome兼容性处理

**示例对话（带线索 - 错误线索）：**

用户：我遇到了划线在不同浏览器看不到的问题

AI：🔍 匹配到 Bug B001，自动尝试方案A（primary）...

用户：不行，控制台报错了："Cannot read property 'cfi' of undefined"

### 场景4：AI尝试成功（用户说"解决了"/"修复了"）

**触发词：** "解决了" / "可以了" / "没问题了" / "修复了" / "好了" / "搞定了" / "成功了" / "验证通过" / "测试通过"

```
用户：修复了 / 解决了 / 可以了

AI自动执行：
1. 🎯 识别当前修复的Bug（从对话上下文）
2. 📝 记录AI尝试成功到 Bug索引.AI尝试历史
   ├── 标记最后尝试的方案为"成功"
   ├── 记录成功时间
   └── 记录成功方案的详细说明
3. 🏷️ 更新方案状态
   ├── 成功方案 → 状态改为"🟢 在用"
   ├── 成功方案 → 优先级改为"⭐ primary"
   └── 其他方案 → 状态改为"🔴 废弃"或保留为备选
4. 📊 更新修复历史
   ├── 添加成功记录到"AI尝试历史"
   └── 更新"解决状态"为"✅ 已解决"
5. 📚 【询问用户】是否更新技能文档
6. 📋 【询问用户】是否创建复盘存档
7. 🎉 生成成功总结报告
```
