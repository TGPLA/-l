
## Nginx 配置

### 配置文件位置

`/opt/1panel/www/conf.d/readrecall.conf`

### 关键配置说明

```nginx
server {
    listen 80;
    listen 443 ssl;
    server_name linyubo.top;
    
    # SSL 证书配置
    ssl_certificate /www/sites/readrecall/ssl/fullchain.pem;
    ssl_certificate_key /www/sites/readrecall/ssl/privkey.pem;
    
    # 前端静态文件目录
    root /www/sites/readrecall/index;
    
    # API 反向代理
    location /api/ {
        proxy_pass http://127.0.0.1:8080/api/;
    }
    
    # 前端路由支持
    location / {
        try_files $uri $uri/ /index.html;
    }
}
```

### 配置要点

- **前端目录**：`/www/sites/readrecall/index`
- **API 代理**：`/api/` → `http://127.0.0.1:8080/api/`
- **HTTPS**：已配置 SSL 证书
- **缓存策略**：
  - `index.html` 禁止缓存
  - 带 hash 的静态资源长期缓存（1年）

## Docker 卷挂载配置

### 为什么要配置卷挂载

EPUB 文件需要持久化存储，容器内的文件系统是临时的，删除容器会导致数据丢失。

### 当前配置

```yaml
volumes:
  - /data/readrecall/uploads:/app/uploads
```

- **容器内路径**：`/app/uploads`
- **服务器路径**：`/data/readrecall/uploads`

### 检查服务器存储目录

```bash
# 查看 EPUB 文件
ssh root@114.132.47.245 "ls -la /data/readrecall/uploads/"
```

### 重新部署后端容器

```bash
# 1. 停止并删除旧容器
docker stop readrecall-backend && docker rm readrecall-backend

# 2. 启动新容器（带卷挂载）
docker run -d --name readrecall-backend \
  --network readrecall-network --network 1panel-network \
  -p 8080:8080 \
  -e SERVER_PORT=8080 \
  -e DB_HOST=1Panel-mysql-OHb5 \
  -e DB_PORT=3306 \
  -e DB_USER=root \
  -e DB_PASSWORD=mysql_h8TCKC \
  -e DB_NAME=reading_reflection \
  -e JWT_SECRET=readrecall-jwt-secret-2024 \
  -e ZHIPU_API_KEY=your-api-key \
  -e ZHIPU_MODEL=glm-4-flash \
  -v /data/readrecall/uploads:/app/uploads \
  --restart unless-stopped \
  readrecall-backend:latest
```

## 教训与注意事项

### 教训：每次部署必须重新构建

**问题描述（2026-04-13）**：
- 只更新了源代码（package.json、public/changelog.json），但直接使用了旧的 dist 目录
- 上传到服务器的仍然是旧版本（1.7.7），导致用户看到旧界面

**教训**：
1. **每次部署前，必须删除旧的 dist 目录**
2. **每次部署前，必须重新运行 `npm run build`**
3. **构建完成后，必须验证本地 dist 目录里的 changelog.json 版本**
4. **特别是修改了 public/ 目录下的文件时（如 changelog.json），这些文件会被复制到 dist 目录，必须重新构建**

**正确的部署流程**：
```powershell
# 1. 删除旧的构建目录
Remove-Item -Recurse -Force dist

# 2. 重新构建
npm run build

# 3. 验证构建产物
cat dist/changelog.json  # 确认版本号正确

# 4. 部署
.\deploy.ps1
```

## 故障排查

### 404 错误

1. 检查文件是否上传到正确目录
2. 检查 Nginx 配置中的 root 路径
3. 检查文件权限

### API 请求失败

1. 检查后端服务是否运行
2. 检查 Nginx 反向代理配置
3. 查看 Nginx 错误日志

### 样式/JS 加载失败

1. 清除浏览器缓存
2. 检查文件是否完整上传
3. 检查 Nginx 缓存配置
